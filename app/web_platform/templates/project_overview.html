ProcSentinel AI - Project Overview

ProcSentinel AI is an AI-enhanced Linux runtime security platform prototype. It analyzes Linux commands before and during execution, calculates risk, applies context-aware policy, records audit evidence, and exposes the results through a dashboard and exportable reports.

Problem

Traditional command filtering can block or allow commands without understanding execution context. ProcSentinel AI distinguishes between safe, sensitive, destructive, and critical command behavior.

Solution

Commands pass through an Execution Security Gateway that analyzes target paths, risk factors, workspace boundaries, rules, and human approval requirements before execution.

Security Decisions

Decision Meaning
allow Low-risk command can run normally.
allow_with_monitoring Command is allowed, but runtime behavior is monitored and recorded.
require_confirmation Command requires human review before execution.
block_critical Command is denied before execution because it targets a critical system path or behavior.

Final Academic Scope

The current version is a strong final academic prototype. It is intentionally below full production-grade security tooling, while remaining suitable for a high-quality final course project.

Future work includes deeper Linux isolation using namespaces, cgroups, and seccomp, stronger authentication, role-based dashboard access, richer AI evaluation metrics, and improved report visualization.
