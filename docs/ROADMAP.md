# Project Roadmap

## Phase 1: Environment Setup

Prepare Ubuntu, Python virtual environment, Git repository, project structure, and required system tools.

Status: Completed

## Phase 2: Architecture Design

Document system architecture, components, data flow, detection pipeline, and demo scenarios.

Status: In Progress

## Phase 3: Sandbox Core

Build the first executable sandbox runner that can start a Linux process, track its PID, and record execution metadata.

Status: Not Started

## Phase 4: Monitoring Engine

Collect CPU usage, memory usage, process status, child processes, and basic runtime events.

Status: Not Started

## Phase 5: Syscall and File Activity Tracing

Use strace to collect syscall-level behavior and detect file-related activity.

Status: Not Started

## Phase 6: Rule-Based Detection

Implement deterministic detection rules and generate an initial risk score.

Status: Not Started

## Phase 7: Database Integration

Store process runs, events, features, alerts, and risk scores.

Status: Not Started

## Phase 8: AI-Based Anomaly Detection

Build a feature dataset and train an initial anomaly detection model.

Status: Not Started

## Phase 9: API Layer

Expose backend endpoints for running processes, listing runs, retrieving events, and showing alerts.

Status: Not Started

## Phase 10: Monitoring Interface

Build a simple dashboard to visualize processes, alerts, and risk scores.

Status: Not Started

## Phase 11: Demo Scenarios

Prepare safe, suspicious, and ransomware-like simulations.

Status: Not Started

## Phase 12: Testing and Validation

Validate sandbox execution, monitoring accuracy, detection logic, API behavior, and demo reliability.

Status: Not Started

## Phase 13: Report and Presentation

Prepare final report, screenshots, diagrams, and live demo sequence.

Status: Not Started
