# Final Presentation Script

## Opening

مشروعنا هو منصة ذكية لعزل وتحليل سلوك العمليات في أنظمة Linux باستخدام تقنيات تعلم الآلة لدعم القرار الأمني.

الفكرة الأساسية أن النظام لا يعتمد على اسم الملف أو التواقيع التقليدية فقط، بل يراقب سلوك العملية أثناء التنفيذ.

## Architecture

يتكون المشروع من ثلاث طبقات رئيسية.

الطبقة الأولى هي طبقة نظم التشغيل. هذه الطبقة مسؤولة عن تشغيل العملية داخل بيئة مراقبة، تطبيق حدود الموارد، مراقبة PID والحالة والذاكرة والمعالج والملفات المفتوحة والعمليات الفرعية، وتتبع استدعاءات النظام باستخدام strace.

الطبقة الثانية هي طبقة الأمن السيبراني. هذه الطبقة تستقبل بيانات المراقبة واستدعاءات النظام، ثم تطبق قواعد كشف أمنية مثل منع الأوامر الخطرة، كشف الوصول إلى الملفات الحساسة، كشف النشاط الشبكي، وكشف الأخطاء أثناء التنفيذ.

الطبقة الثالثة هي طبقة تحليل السلوك باستخدام تعلم الآلة. هذه الطبقة لا تستبدل القواعد الأمنية، بل تساعدها من خلال إنتاج anomaly score يدعم القرار النهائي.

## Execution Flow

عند إدخال أمر إلى النظام، لا يتم تشغيله مباشرة. أولًا يمر عبر Command Policy. إذا كان خطرًا يتم منعه قبل التنفيذ. إذا كان مسموحًا، يتم تشغيله ضمن Sandbox Runner مع حدود للوقت والذاكرة والمعالج وعدد الملفات المفتوحة.

أثناء التشغيل، يقوم Process Monitoring Engine بجمع عينات عن العملية. وبالتوازي يتم استخدام strace لتتبع استدعاءات النظام. بعد ذلك يتم تلخيص البيانات، استخراج الخصائص السلوكية، تطبيق قواعد الكشف، ثم استخدام طبقة تعلم الآلة لإنتاج درجة شذوذ.

في النهاية ينتج النظام قرارًا واضحًا: allow أو review أو block_or_investigate.

## Demo Scenario 1

السيناريو الأول هو عملية آمنة. النتيجة المتوقعة أن تكتمل العملية بدون قواعد خطرة، ويكون القرار النهائي allow.

## Demo Scenario 2

السيناريو الثاني هو محاولة الوصول إلى مسار حساس مثل /etc/passwd. هنا لا نعتبر السلوك مدمّرًا مباشرة، لكنه يحتاج مراجعة، لذلك القرار النهائي review.

## Demo Scenario 3

السيناريو الثالث هو أمر خطر مثل rm -rf. يتم منعه قبل التنفيذ بواسطة Command Policy، والقرار النهائي block_or_investigate.

## AI Explanation

طبقة تعلم الآلة في المشروع هي طبقة مساعدة لتحليل السلوك غير المعتاد. لا ندعي أنها تكشف كل البرمجيات الخبيثة، بل تعطي anomaly score يساعد طبقة القرار النهائي مع قواعد الأمن السيبراني.

## Closing

قيمة المشروع أنه يجمع بين مفاهيم نظم التشغيل، الأمن السيبراني، وتحليل السلوك بتعلم الآلة ضمن نموذج أكاديمي قابل للتطوير.
